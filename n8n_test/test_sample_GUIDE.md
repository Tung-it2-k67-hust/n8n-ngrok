## SECTION 1: CORE MENTAL MODEL

### Core Definitions

**Java (Java)**: Ngôn ngữ lập trình hướng đối tượng (Object-Oriented Programming - OOP) được sử dụng rộng rãi để xây dựng các ứng dụng đa nền tảng. Trong ngữ cảnh tài liệu này, nó được dùng để in ra màn hình console.

**Method (Phương thức)**: Một khối mã được đặt tên thực hiện một tác vụ cụ thể. Trong ví dụ `System.out.println()`, `println` là một phương thức.

**System.out.println()**: Lệnh chuẩn của Java để xuất dữ liệu (typically text) ra luồng вывод chuẩn (standard output), thường là cửa sổ console hoặc terminal.

### Why this approach is better than others
Việc sử dụng `System.out.println()` là phương pháp cơ bản nhất để giao tiếp với lập trình viên hoặc người dùng trong môi trường console. Nó đơn giản, dễ kiểm tra và không yêu cầu cấu hình phức tạp so với việc ghi log vào file hay xây dựng GUI ngay từ đầu.

**Code Demo:**

```java
public class HelloWorld {
    public static void main(String[] args) {
        // Xuất ra chuỗi ký tự "Hello World"
        System.out.println("Hello World");
        
        // Xuất ra giá trị biến
        int version = 8;
        System.out.println("Java Version: " + version);
    }
}
```

## SECTION 4: CODE PATTERNS

### Pattern: Standard Entry Point
**Khi nào dùng (When to use):** Khi bắt đầu một chương trình Java mới hoặc viết một bài tập đơn giản.
**Tại sao đúng (Why this pattern is correct):** Đây là điểm bắt đầu (Entry Point) mà Java Virtual Machine (JVM) gọi khi chạy chương trình. Thiếu nó, chương trình sẽ không thể chạy được.

**Code Example (Full):**

```java
// Tên file phải trùng với tên class: HelloWorld.java
public class HelloWorld {
    
    // Đây là phương thức chính - điểm bắt đầu执行
    public static void main(String[] args) {
        
        // Logic: In ra màn hình
        System.out.println("Hello World");
        
        // Logic: In nhiều dòng
        System.out.println("Dòng 1");
        System.out.println("Dòng 2");
    }
}
```

## SECTION 5: ANTI-PATTERNS & WARNINGS

1.  **In quá nhiều log (Excessive Logging)**
    *   **Lý do nguy hiểm:** Sử dụng `System.out.println()` hàng ngàn lần trong một vòng lặp lớn có thể làm treo console và giảm hiệu năng nghiêm trọng. Nó không có mức độ ưu tiên (levels) như log của hệ thống thật (INFO, ERROR, WARN).
    *   **Cách tránh:** Chỉ dùng cho debug nhỏ. Dùng thư viện log (như Log4j hoặc SLF4J) cho项目 lớn.

2.  **Lỗi biên dịch (Compilation Error)**
    *   **Lý do nguy hiểm:** Thiếu dấu chấm phẩy `;` ở cuối câu lệnh.
    *   **Ví dụ sai:** `System.out.println("Hello World")`
    *   **Ví dụ đúng:** `System.out.println("Hello World");`

3.  **Lỗi nhập sai cấu trúc Class/Main**
    *   **Lý do nguy hiểm:** Java yêu cầu cú pháp nghiêm ngặt. Nếu tên file là `Test.java` thì bên trong phải có `public class Test`.
    *   **Hậu quả:** Lỗi `NoSuchMethodError` hoặc `ClassNotFoundException`.

## SECTION 6: MASTER CHEAT SHEET

### Quick Reference Rules
1.  **Cú pháp:** `System.out.println("Nội dung");`
2.  **Dấu nháy:** Dùng dấu ngoặc kép `""` cho chuỗi text.
3.  **Kết thúc:** Luôn kết thúc câu lệnh bằng dấu chấm phẩy `;`.

### Decision Logic
*   **Nếu muốn in xuống dòng:** Dùng `println()`.
*   **Nếu muốn in không xuống dòng:** Dùng `print()`.
*   **Nếu muốn in biến:** Dùng dấu `+` để nối chuỗi (e.g., ` "Age: " + age`).

### Top 3 Things to Remember
1.  `public static void main(String[] args)` là bắt buộc để chạy code.
2.  `System` là class, `out` là đối tượng, `println` là phương thức.
3.  Java là ngôn ngữ phân biệt hoa thường (Case-sensitive): `system.out` là sai, `System.out` là đúng.
